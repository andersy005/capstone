    \textbf{An update regarding our project:}

Project requirements:

\begin{itemize}
\tightlist
\item
  Pulse Width Modulation
\item
  Regulate repetition rate for laser driver.
\end{itemize}

P.S: When the laser is running in \textbf{pulsed mode} the number of
pulses per second or frequency of pulses is known as the \textbf{Pulse
Repetition rate (PRR)}. The term PRR and pulse frequency are commonly
used.

Status of development:

\begin{itemize}
\tightlist
\item
  We are using MicroPython, which is an implementation of Pyton Language
  written in C, that is optimized to run on microcontrollers.
\item
  We've been able to generate a square-wave using the STM32F4 DAC
\item
  The DAC is operating in \textbf{triggered mode}: this is allowing the
  updates to be initiated by a rising edge on an internal or external
  synchronization signal.
\end{itemize}

Still left to be done:

\begin{itemize}
\tightlist
\item
  Using RC components as a low pass filter to remove noise from
  generated signal
\end{itemize}

    What's happening?:

\begin{itemize}
\item
  Initiates a burst of RAM to DAC using a DMA transfer. The input data
  is treated as an array of bytes in 8-bit mode, and an array of
  unsigned half-words (array typecode `H') in 12-bit mode.
\item
  \texttt{freq} can be an integer specifying the frequency to write the
  DAC samples at, using \texttt{Timer(6)}. Or it can be an
  already-initialised Timer object which is used to trigger the DAC
  sample. Valid timers are 2, 4, 5, 6, 7 and 8.
\item
  \texttt{Timer(6)}:
\item
  0 channels - no direct external pin control.
\item
  must be Master
\item
  up count only with DMA control
\item
  16 bit prescaler
\item
  used for timebase timers or triggering the DAC
\item
  max speed 84 MHz, max interface speed 42 MHz
\item
  \texttt{mode} can be \texttt{DAC.NORMAL} or \texttt{DAC.CIRCULAR}
\end{itemize}
